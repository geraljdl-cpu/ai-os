# AI-OS Components

## Overview
AI-OS (Artificial Intelligence Operating System) is designed to manage and automate various tasks within a networked environment. It consists of several components that work together to provide robust functionality.

## Components
The following are the key components of AI-OS:

- **Express**: A web server running on port 3000, serving as an entry point for HTTP requests.
- **Agent-Router**: Manages communication and routing between different agents within the system. Runs on port 5679.
- **Postgres**: The primary database management system used to store data. Operates on port 5432.
- **Modbus**: A protocol for communicating with industrial electronic devices, running on port 5020.
- **Art-Net**: A network protocol designed for the control of lighting and other stage equipment, operating on port 6454.

## Autopilot Flow
The autopilot flow describes how tasks are processed within AI-OS:

1. Tasks start in a backlog queue.
2. They are then picked up by a worker component.
3. The worker processes these tasks using the tools engine.
4. Finally, the results or data are stored in Postgres.

## Operation Commands
The following commands are used to manage and interact with AI-OS:

- `aiosctl`: A command-line utility for controlling various aspects of AI-OS.
- `systemctl --user`: Used to start, stop, restart, or check the status of user services within AI-OS.
- `journalctl`: For viewing logs generated by AI-OS components.

## Getting Started
To get started with AI-OS, follow these steps:

1. Ensure all necessary dependencies are installed.
2. Start each component using the appropriate command (e.g., `systemctl --user start <component>`).
3. Use `aiosctl` to manage tasks and configurations within the system.

For more detailed instructions, refer to the documentation or specific component README files.
